# Тестовое задание: Semantic Search Engine для навигационных данных беспилотного тягача

## Описание проекта

Разработать микросервис (REST API) на Python, который принимает запросы на естественном языке, анализирует их с помощью LLM (OpenRouter, бесплатные модели) и возвращает релевантные данные из набора Parquet/CSV-файлов с навигационной информацией.

Данные представляют собой высокоточную GNSS/INS-телематику (формат NovAtel) и содержат информацию о положении, скорости, ориентации и статусе решения.

**Примечание:** Данные хранятся без timezone. Для временного анализа использовать Moscow UTC+3.

---

## Исходные данные

3 набора данных в формате CSV со следующей структурой:


| Поле | Тип | Описание |
|------|-----|----------|
| `_id` | `Utf8` | Уникальный идентификатор записи |
| `_bag_id` | `Utf8` | Идентификатор ROS bag-файла, из которого получены данные |
| `_timestamp` | `Int64` | Временная метка (Unix timestamp) |
| `_topic` | `Utf8` | Название ROS-топика |
| `_type` | `Utf8` | Тип ROS-сообщения |
| `header__stamp__sec` | `Int32` | Секунды временной метки заголовка |
| `header__stamp__nanosec` | `Uint32` | Наносекунды временной метки заголовка |
| `header__frame_id` | `Utf8` | Идентификатор системы координат |
| `nov_header__message_name` | `Utf8` | Название сообщения NovAtel |
| `nov_header__message_id` | `Uint16` | Идентификатор сообщения NovAtel |
| `nov_header__message_type` | `Uint8` | Тип сообщения NovAtel |
| `nov_header__sequence_number` | `Uint32` | Порядковый номер сообщения |
| `nov_header__time_status` | `Uint8` | Статус времени (GPS, UTC и т.д.) |
| `nov_header__gps_week_number` | `Uint16` | Номер GPS-недели |
| `nov_header__gps_week_milliseconds` | `Uint32` | Миллисекунды внутри GPS-недели |
| `nov_header__receiver_status` | `Uint32` | Битовая маска статуса приемника |
| `nov_header__idle_time` | `Uint32` | Время простоя приемника (мс) |
| `latitude` | `Float64` | Широта (градусы) |
| `longitude` | `Float64` | Долгота (градусы) |
| `height` | `Float64` | Высота над эллипсоидом (метры) |
| `north_velocity` | `Float64` | Северная составляющая скорости (м/с) |
| `east_velocity` | `Float64` | Восточная составляющая скорости (м/с) |
| `up_velocity` | `Float64` | Вертикальная составляющая скорости (м/с) |
| `roll` | `Float64` | Угол крена (градусы) |
| `pitch` | `Float64` | Угол тангажа (градусы) |
| `azimuth` | `Float64` | Азимут/курс (градусы от севера) |
| `pos_type__type` | `Uint8` | Статус качества позиционирования (19 = плохо, 53/54/56 = фикс) |

## Требования к реализации

### Функциональные требования

Реализовать HTTP-сервис с единственным эндпоинтом:

**POST `/query`**

Тело запроса (JSON):
```json
{
  "query": "текст запроса на естественном языке"
}
```

Сервис должен:
1. Проанализировать запрос с помощью LLM через OpenRouter (использовать **бесплатные модели**, например `mistralai/mistral-7b-instruct`, `google/gemma-2-9b-it` или `meta-llama/llama-3-8b-instruct`)
2. Преобразовать запрос в конкретное действие с данными
3. Выполнить необходимые вычисления и фильтрацию на CSV-файлах
4. Вернуть результат в формате JSON

### Поддерживаемые типы запросов

Система должна корректно обрабатывать следующие 5 типов запросов:

| № | Запрос | Тип |
|---|--------|-----|
| 1 | "Какая была максимальная скорость?" | Агрегация |
| 2 | "Покажи все моменты, когда качество позиционирования было плохим" | Фильтрация |
| 3 | "Получить положение тягача в сумерках" | Временной срез |
| 4 | "Найди участки, где тягач резко тормозил (отрицательное ускорение > 2 м/с²)" | Аналитика |
| 5 | "Когда тягач находился в пределах трассы М11 (Россия)?" | Геопространственный фильтр |

**Важно:** Система не должна быть жестко закодирована только на эти 5 запросов. LLM должен самостоятельно классифицировать похожие формулировки (например, "максималка", "top speed", "где были проблемы с gps" и т.д.).

### Вычисляемые метрики

Для корректной работы необходимо реализовать следующие производные поля:

```python
from datetime import datetime, timedelta

# Горизонтальная скорость (м/с)
horizontal_speed = sqrt(north_velocity² + east_velocity²)

# Ускорение (м/с²) — производная скорости по времени
acceleration = diff(horizontal_speed) / diff(timestamp)

# Преобразование timestamp в часы для определения сумерек (с учетом Moscow UTC+3)
hour = (datetime.fromtimestamp(timestamp) + timedelta(hours=3)).hour
```

### Интерпретация нечетких формулировок

| Формулировка | Интерпретация |
|--------------|---------------|
| "Плохое качество позиционирования" | `pos_type__type == 19` |
| "Сумерки" | Время между 16:00 и 19:00 (по Moscow UTC+3) |
| "Резко тормозил" | Отрицательное ускорение < -2 м/с² |
| "Трасса М11" | Координаты: широта 55.5–60.0, долгота 30.0–37.5 |

## Технические требования

### Стек технологий
- **Язык:** Python 3.10+
- **Web-фреймворк:** FastAPI или Flask
- **Работа с данными:** Pandas или Polars
- **HTTP-клиент:** httpx или aiohttp (для запросов к OpenRouter)
- **Контейнеризация:** Docker, docker-compose

### Интеграция с OpenRouter
- Использовать **бесплатные модели** (список актуальных моделей: https://openrouter.ai/docs#models)
- Ключ API кандидат получает самостоятельно
- Реализовать обработку ошибок API (таймауты, rate limits, некорректные ответы)
